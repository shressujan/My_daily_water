Abstract

This article talks about the importance of conserving water resources and the ways we start conserving. It talks about using a database application “My Daily Water Tracker” on a daily basis to help keep track of daily water consumption per person, city, and state. It then talks about why this app is long overdue. The article also talks about logging every water habit in the app daily to help generate an accurate report/statistic which could later be used for research purpose. The main goal of this project is to get a greater number of people use this app and hopefully improve their water habits and, in the process, create a positive impact water resources conservation.
Keywords—daily water, habits, app, MySQL, report

Introduction

Water, water everywhere but not a drop to drink. This has become a very popular proverb in past few years. About 71 percent of the Earth's surface is water-covered, and the oceans hold about 96.5 percent of all Earth's water [1]. That seems like a lot of water, right? While this is true, but we also can’t ignore the fact that only a few percentages of the water that’s on the earth surface is actually usable. Of the waters occupying 70% of the earth’s surface, only 3% is considered fresh water. Furthermore, about 2.6% of this freshwater is inaccessible for humans.” So, this only leaves us approximately 0.4% of the earth’s water which is actually usable for drinking and household purposes, and that percentage has to be shared among the 7 billion people in earth [2]. And to make it even worse, the amount usable water is declining while the population around the world is increasing at a rapid rate and this makes it very hard to meet the water supply needs of the ever-growing population. This is why many countries, over the past few years, have already begun to face water supply shortage problems.
The project initially started out with an idea about a web-based application that would track user’s daily water usage, and therefore the name of the project “My Daily Water Tracker”. Every big database must compulsorily have ER diagram to show the flow of the data. This project is no different, it has an ER diagram that shows the relationship between 4 main entities in the database. i.e. User_Account that holds information about all the active user accounts, User that hold information about the user themselves, Address that hold information about user addresses, and last but not least Daily_Water_Usage that keeps record of all the major water usages per user.
The purpose of this project is to design an awareness application that educate its’s users about their daily water habits and help them realize how much water they are wasting on a daily basis. And now since, the project is fully functional, the app definitely going to make some positive impact on the people and the environment. The general overview of how this app works is, it records the daily water usage inputs from the users, analyzes the data, and finally generate a report which is then presented to the user in a graphical form to help them better understand about their water usage habits.
Like every good application has strong foundations, this app also has two very strong foundation that carries the entire application. And, they are MYSQL database and python. The app uses python programming language for the user interface/front-end, and MYSQL database for the backend or storing the data. The user interface is kept very simple, there is a website that users will connect to, a welcome page, a sign-up page, a login page, an input page, and last but not least a report page. The welcome page hosts the information about the application itself and its intended use. The login page allows the active users to login in and start using the app. The sign-up page allows new users create an account in the centralized MYSQL database. After the user has logged in, the input page is pulled up. The input page allows the users to log their daily water habits such as shower, kitchen, bathroom, drinking water, miscellaneous etc. And for that, input page provides users sliders which they can use to select the appropriate amount of water used for each category. For example:- there is a kitchen sink slider that tracks water used in kitchen sink per day, there is a shower slider to track the amount of water used during shower, there is also a drinking water slider to keep track of the number of cups of water drank in a day, and there is bathroom sink slider that keeps track of water used in bathroom sink, and so on. Users have the option to view their water habits on the report page. The main purpose of this page is to help create an impact on the users and improve their water habits. Additionally, users can also view the water report of a city or a state. And this is very helpful for research purpose or if someone is just curious to know which city is using excess amount of water, or which city has decreased the amount of water used after its people started using the app, and same goes for different states.

Python for Front-end development

This project uses python as a principal programming language. Python is an interpreted, high-level, general purpose programming language to help programmers write clear, logical codes for small or large-scale projects.[3] The reason to choose Python for front-end development is because it is a highly flexible programming language, meaning python supports a vast number of frameworks and has thousands of libraries that can be imported to designed and perform very specific tasks. In this project, python uses a micro-framework called Flask. Flask is a very popular python framework designed for specifically for front-end development. In this project, Flask is not only being used for developing the front-end/user interface but also being used for importing several other libraries such as session, encryption, jsonify, json, HTML etc. Also being used in the project is MYSQL library. This library basically creates a bridge between the front-end such as website and the backend such as the MYSQL database. This library is very critical in this application as without it, the front-end and backend would not be able to connect with each other and act as separate entities.

MYSQL for Back-end development

MYSQL is a highly popular relational database management system. Its name is a combination of “My”, the name of co-founder Michael Widenius’s daughter, and “SQL”, the abbreviation for Structured Query Language.[4] It is very commonly used in different industries to store data. This project also uses MYSQL to store the data. MYSQL Workbench, a tool to communicate with MYSQL server, is also being used in the project for the intended purpose. The MYSQL database has been separated in 4 different entities with specific relationship to each other. The four entities are User_Account, User, Address, and Daily_Water_Usage. User_Account, as mentioned before stores all the active user accounts. User entity holds data about the active users such as first name, last name etc. Address entity retains the information about user addresses. And, finally the Daily_Water_Usage entity is where key data such as shower_data, kitchen_sink_data, bathroom_sink_data, toilet_data, sprinkler_data etc. are stored. This is the only entity that get updated regularly as per users input. The data from this table is very crucial for this project since it’s used for data analysis and data visualization which is the core of this project.
